VPN Access

To access company-internal resources, you must first connect to the Guppy VPN. Your corporate device should already have the necessary client software installed. You may need to run the VPN software as a privileged user.

During the connection process, the client software will ask you to specify your VPN credentials. Use the same username and password that you used to log into this page.

Once you are connected to the Guppy VPN, you should be able to reach internal systems via their IP addresses.

File Storage

Corporate IT provides the FTP server for business use only. Your VPN credentials can be used to authenticate to the FTP server at 10.0.0.2 or fd00:ec5c::2.

Frequently Asked Questions

How do I upload files to the FTP server

Follow this guide to get started with using FTP software to copy files between your machine and the FTP server.

I cannot connect to the VPN

Verify that your credentials are correct and that you are connecting to the correct server ({{ host }}, on port 9100). If you are still facing issues, contact IT support. Please note that IT support will never ask you for your password; do not share it with anyone.

Where can I get help

If you encounter issues with this network device, please open a ticket with corporate IT. Remember to follow the company-wide information disclosure policies at all times.

Administration

To diagnose networking issues, you are able to see all dropped network packets sent by your VPN connection on the Dropped Traffic page. Note that some traffic may not be allowed by firewall rules.

This Guppy Networks firewall device also exposes an SNMP endpoint. You can monitor the status of the device using the Monitoring interface, and monitor user specific values using the User Monitoring overview. Configuration of organization-specific SNMP fields is possible on the Custom Values page.

The Advanced interface is intended for use by trained and authorized personnel only. Use of this interface requires prior approval from corporate IT.
